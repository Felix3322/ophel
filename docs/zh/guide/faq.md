# 常见问题

## 安装相关

### 为什么扩展不能在应用商店安装？

Ophel 目前正在各应用商店审核中，暂时请通过 [GitHub Releases](https://github.com/urzeye/ophel/releases) 下载安装。

### 安装后扩展不生效？

1. 确认已在扩展管理页面启用 Ophel
2. 刷新目标网页（`Ctrl/Cmd + R`）
3. 检查是否与其他扩展冲突
4. 尝试禁用其他 AI 相关扩展后重试

### Firefox 临时加载后扩展消失了？

Firefox 临时加载的扩展在浏览器关闭后会失效，这是 Firefox 的安全限制。解决方案：

- 等待扩展上架 Firefox Add-ons
- 使用 Firefox 开发者版，支持安装未签名扩展

## 功能相关

### 大纲不显示？

1. 确认当前页面是支持的 AI 平台
2. 查看大纲面板是否被折叠（点击边缘展开）
3. 检查设置中是否禁用了大纲功能
4. 刷新页面重试

### 自定义 CSS 不生效？

1. 确认 CSS 语法正确（无拼写错误）
2. 检查选择器是否匹配目标元素
3. 尝试添加 `!important` 提升优先级
4. 使用浏览器开发者工具检查样式覆盖情况

### 快捷键不工作？

1. 确认快捷键没有与浏览器或系统冲突
2. 焦点需要在网页内容上（非地址栏等）
3. 尝试在设置中重新配置快捷键
4. macOS 用户注意：`Alt` 对应 `Option` 键

### WebDAV 备份失败？

1. 检查 WebDAV 服务器地址格式是否正确
2. 确认用户名和密码无误
3. 检查网络连接
4. 确认已授予 `<all_urls>` 权限
5. 坚果云用户：使用应用专用密码而非登录密码

### AI Studio 界面显示异常？

AI Studio 界面较为特殊，如遇问题：

1. 尝试刷新页面
2. 检查是否启用了「自动折叠导航栏」
3. 尝试调整宽屏模式设置

## 隐私相关

### Ophel 会收集我的对话数据吗？

**不会**。Ophel 完全在本地运行：

- 不会上传任何对话内容
- 不会收集用户行为数据
- 不会与第三方共享信息

### 数据存储在哪里？

| 数据        | 位置                            |
| ----------- | ------------------------------- |
| 本地数据    | 浏览器 LocalStorage / IndexedDB |
| 本地备份    | 你选择的下载位置                |
| WebDAV 备份 | 你指定的 WebDAV 服务器          |

### 清除浏览器数据会丢失设置吗？

是的。清除浏览器数据（包括 LocalStorage 和 IndexedDB）会导致 Ophel 数据丢失。

**建议**：

- 定期使用「导出」功能备份数据
- 或配置 WebDAV 自动同步

## 其他问题

### 如何报告 Bug？

1. 前往 [GitHub Issues](https://github.com/urzeye/ophel/issues)
2. 点击「New Issue」
3. 选择「Bug Report」模板
4. 填写问题详情，包括：
   - 浏览器版本
   - 扩展版本
   - 问题复现步骤
   - 预期行为与实际行为

### 如何请求新功能？

1. 前往 [GitHub Issues](https://github.com/urzeye/ophel/issues)
2. 搜索是否已有类似请求
3. 如没有，点击「New Issue」
4. 选择「Feature Request」模板
5. 详细描述你的需求和使用场景
